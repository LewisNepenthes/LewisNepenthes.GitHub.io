Markdown写作时的图片大小问题

2020-10-20_17-36-52

前言

在md写作时，经常会遇到图片展示时过大的问题，那么改如何随心所欲地修改图片的大小呢？下面就探索的几个方法总结一下。

方法

全局修改

在css文档中使用zoom

img { zoom: 50%; }

视觉问题

由于不同编辑器设置不一样，就拿我用的typora来举例吧：
默认的主题最窄也有860px，减去留白也有800px，效果是这样的：
图片占了大部分画面，下面的内空都看不到了。 调成600px后，就好多了：

写法1

#write{
max-width: 660px; /* 原为860px; */ }

写法2

img { max-width: 80%; }

写法3

padding-left = (100% - width) / 2

自动半全局修改

在图床链接后在线压缩

imageView2

Markdown 不支持 by design，可以直接用 html 或者 找一个支持参数的图床，比如七牛。 例子：原图http://siwangxinyuan.qiniudn.com/jianshu-42-1280x800-%5BDesktopNexus.com%5D.jpg 指定尺寸http://siwangxinyuan.qiniudn.com/jianshu-42-1280x800-%5BDesktopNexus.com%5D.jpg?imageView/2/w/619/q/90

imageMorgr2

使用七牛:下面是图片居中并以原尺寸一般的形式显示.

::: {align=”center”}

:::

更多信息请看这里:高级处理（imageMogr2）

局部修改

CSS Inline Attribute

改写为{:class=”img-responsive”} {:height=”50%” width=”50%”} {:height=”100px” width=”400px”} Kramdown的Inline Attribute，直接在图片后面加上对应的CSS样式即可

-w number

→ 修改后代码（加粗部分为新增代码，数字为缩放大小）：修改任意值即可改变图片尺寸。

CSS Attribute Selector

img: img[src*=”#width-full”] { width: 100%;} md: 间接为图片加上一个 class，该技能可叠加使用。如果喜欢，你当然可以在 stylesheet 里给每个尺寸写上一条。

Space=法

注意 =前有个空格，可以只写宽度，Mou中支持

Typora右键法

右键Typora里的图片，选择img的zoom调整比例。 最好用的方法，随心所欲。
